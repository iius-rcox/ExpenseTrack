# Status Workflow

Understand report status and the approval process.

## Overview

Expense reports move through a status workflow from creation to approval. Each status indicates where the report is in the process.

## Report Statuses

![Report status cards](../../images/reports/report-status-cards.png)
*Caption: Reports in various status states*

### Draft

**Meaning**: Report created but not yet submitted

**What you can do**:
- Edit report notes
- Review included expenses
- Delete the report
- Submit for approval

**Icon**: Pencil or document outline

### Submitted

**Meaning**: Sent for manager/approver review

**What you can do**:
- View report contents
- Check submission time
- Wait for approval
- Withdraw (if allowed)

**Icon**: Arrow or send icon

### Approved

**Meaning**: Manager approved the report

**What you can do**:
- View final report
- Export for records
- Report is complete

**Icon**: Checkmark

### Rejected

**Meaning**: Manager returned the report for changes

**What you can do**:
- View rejection reason
- Make requested changes
- Resubmit when ready

**Icon**: X or return arrow

## Status Flow

```
Created → Draft → Submitted → Approved
                      ↓
                  Rejected
                      ↓
             (Make changes)
                      ↓
                 Resubmit → Approved
```

## Submitting a Report

### Prerequisites

Before submitting:
- Report is in Draft status
- All expenses are verified
- Notes are added (if needed)

### Submit Steps

1. Open the draft report
2. Click **Submit** or **Submit for Approval**
3. Optionally add a message for the approver
4. Confirm submission
5. Status changes to **Submitted**

### What Happens After Submission

1. Approver receives notification
2. They review expenses and receipts
3. They approve or reject
4. You receive notification of decision

## When Reports Are Rejected

### Understanding Rejection

If a report is rejected:

1. You receive a notification
2. The report status shows **Rejected**
3. A rejection reason is provided
4. The report returns to editable state

### Common Rejection Reasons

| Reason | Action |
|--------|--------|
| Missing receipt | Upload and match the missing receipt |
| Wrong category | Recategorize the expense |
| Needs explanation | Add notes explaining the expense |
| Amount error | Verify and correct the amount |
| Duplicate | Remove duplicate entries |

### Fixing Rejected Reports

1. Open the rejected report
2. Read the rejection notes
3. Make requested changes:
   - Add missing receipts/matches
   - Edit notes or categories
   - Correct errors
4. Resubmit when fixed

## Tracking Report Status

### Reports List View

The Reports page shows all your reports with:
- Report name/date
- Status indicator (color-coded)
- Total amount
- Submission date
- Last action date

### Filtering by Status

Filter the list to show only:
- Draft reports
- Submitted reports
- Approved reports
- Rejected reports

### Notifications

You receive notifications when:
- Report is approved
- Report is rejected
- Approver requests information

## Timeline View

Some reports show a timeline:
- Creation date
- Submission date
- Approval/rejection date
- Any comments or notes
